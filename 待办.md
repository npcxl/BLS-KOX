可以，那 sys_storage_config 就不要只按 MinIO 设计，而是设计成 通用对象存储配置表，支持：

minio
aliyun_oss
tencent_cos
qiniu_kodo
huawei_obs
aws_s3
local

核心原则是：通用字段单独存，厂商差异字段放 JSON。

一、推荐表结构：sys_storage_config
CREATE TABLE sys_storage_config (
  storage_id VARCHAR(32) PRIMARY KEY COMMENT '存储配置ID',

  tenant_id VARCHAR(32) NOT NULL DEFAULT '000000' COMMENT '租户ID，000000表示平台默认配置',

  storage_name VARCHAR(100) NOT NULL COMMENT '存储名称',
  storage_type VARCHAR(30) NOT NULL COMMENT '存储类型：minio/aliyun_oss/tencent_cos/qiniu_kodo/huawei_obs/aws_s3/local',

  endpoint VARCHAR(255) DEFAULT NULL COMMENT '访问端点，不带协议，例如 s3.minion.xlcig.cn / oss-cn-hangzhou.aliyuncs.com',
  region VARCHAR(100) DEFAULT NULL COMMENT '地域，例如 oss-cn-hangzhou / ap-shanghai',
  port INT DEFAULT NULL COMMENT '端口，MinIO常用443/9000',
  use_ssl TINYINT NOT NULL DEFAULT 1 COMMENT '是否使用HTTPS',

  access_key VARCHAR(255) DEFAULT NULL COMMENT 'AccessKey，可加密',
  secret_key TEXT DEFAULT NULL COMMENT 'SecretKey，必须加密',

  public_bucket VARCHAR(100) DEFAULT NULL COMMENT '公共桶',
  private_bucket VARCHAR(100) DEFAULT NULL COMMENT '私有桶',

  public_base_url VARCHAR(500) DEFAULT NULL COMMENT '公共访问基础URL',
  private_base_url VARCHAR(500) DEFAULT NULL COMMENT '私有访问基础URL，一般可为空',

  path_style TINYINT NOT NULL DEFAULT 0 COMMENT '是否Path Style访问，MinIO通常为1，云厂商通常为0',

  config_json JSON DEFAULT NULL COMMENT '厂商差异配置JSON',
  policy_json JSON DEFAULT NULL COMMENT '上传限制/访问策略JSON',

  is_default TINYINT NOT NULL DEFAULT 0 COMMENT '是否默认配置',
  status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1启用，0停用',

  remark VARCHAR(500) DEFAULT NULL,

  create_by VARCHAR(32) DEFAULT NULL,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_by VARCHAR(32) DEFAULT NULL,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted TINYINT NOT NULL DEFAULT 0,

  INDEX idx_tenant_id (tenant_id),
  INDEX idx_storage_type (storage_type),
  INDEX idx_status (status),
  INDEX idx_default (is_default)
) COMMENT='对象存储配置表';
二、为什么要加 config_json？

因为不同厂商字段不一样。

比如 MinIO 需要：

{
  "forcePathStyle": true
}

阿里云 OSS 可能需要：

{
  "internalEndpoint": "oss-cn-hangzhou-internal.aliyuncs.com",
  "cname": false,
  "timeout": 60000
}

腾讯云 COS 可能需要：

{
  "appId": "1250000000",
  "bucketSuffixWithAppId": true,
  "protocol": "https:"
}

七牛云可能需要：

{
  "zone": "Zone_z0",
  "domain": "https://cdn.xxx.com"
}

所以不要给每个厂商都加一堆字段，不然表会越来越乱。

八、后端代码结构建议

你后端不要在业务里直接判断各种厂商，做成适配器模式。

src/modules/storage/
├── storage.controller.ts
├── storage.service.ts
├── storage.repository.ts
├── providers/
│   ├── StorageProvider.ts
│   ├── MinioProvider.ts
│   ├── AliyunOssProvider.ts
│   ├── TencentCosProvider.ts
│   ├── AwsS3Provider.ts
│   └── LocalProvider.ts
└── storage.factory.ts

统一接口：

export interface StorageProvider {
  upload(params: UploadParams): Promise<UploadResult>;
  remove(params: RemoveParams): Promise<void>;
  getPublicUrl(params: UrlParams): string;
  getPrivateUrl(params: UrlParams): Promise<string>;
  ensureBucket?(bucketName: string): Promise<void>;
}

工厂根据 storage_type 返回不同实现：

export function createStorageProvider(config: StorageConfig): StorageProvider {
  switch (config.storageType) {
    case 'minio':
      return new MinioProvider(config);
    case 'aliyun_oss':
      return new AliyunOssProvider(config);
    case 'tencent_cos':
      return new TencentCosProvider(config);
    case 'aws_s3':
      return new AwsS3Provider(config);
    case 'local':
      return new LocalProvider(config);
    default:
      throw new Error(`不支持的存储类型：${config.storageType}`);
  }
}
九、多租户优先级

查询配置时建议这样：

优先使用当前租户自己的配置
没有则使用平台默认配置 tenant_id = 000000

SQL：

SELECT *
FROM sys_storage_config
WHERE deleted = 0
  AND status = 1
  AND (tenant_id = :tenantId OR tenant_id = '000000')
ORDER BY
  CASE WHEN tenant_id = :tenantId THEN 0 ELSE 1 END,
  is_default DESC,
  create_time DESC
LIMIT 1;

这样你后期可以做到：

普通租户：使用平台 MinIO
大客户 A：使用自己的阿里云 OSS
大客户 B：使用自己的腾讯云 COS
平台公共文件：使用平台默认 MinIO
十、我的最终建议

你这个表就按 “通用字段 + config_json 扩展字段 + policy_json 策略字段” 设计。

最终架构是：

sys_storage_config
负责存：
- 存储类型
- endpoint
- accessKey / secretKey
- 公共桶 / 私有桶
- 公共URL
- 厂商扩展配置
- 上传策略

sys_file
负责存：
- 文件归属租户
- 使用哪个存储配置
- bucket
- objectName
- public/private
- 文件元信息

这样以后你要接阿里云、腾讯云、MinIO、本地存储，都不用改表，只需要新增一个 Provider 实现。
