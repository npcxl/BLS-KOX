不要马上拆成真正微服务。请先为 BLS-KOX 做“分布式能力预留 + 模块化单体增强”。

目标：
1. Koa 和 Java 后端都保持单体可独立启动。
2. 不修改前端业务代码。
3. 不引入注册中心、网关、Nacos、Seata。
4. 先实现分布式基础能力：
   - requestId / traceId
   - Redis 缓存
   - 分布式锁
   - 幂等 Key
   - Redis 限流
   - 异步任务队列
   - Metrics 指标
   - 健康检查
5. 文档新增微服务路线，但不要实际拆服务。

Java 侧：
- 新增 distributed 包：
  com.bls.server.distributed.lock
  com.bls.server.distributed.idempotent
  com.bls.server.distributed.ratelimit
  com.bls.server.distributed.trace
- 增加 @DistributedLock
- 增加 @Idempotent
- 增加 @RateLimit
- 增加 TraceFilter，写入 MDC：requestId、traceId、tenantId、userId
- Redis 作为锁、限流、幂等存储
- 增强 Actuator/Micrometer 指标

Koa 侧：
- 增加 src/distributed/lock.ts
- 增加 src/distributed/idempotency.ts
- 增加 src/distributed/rate-limit.ts
- 增加 trace/request context
- Redis 作为锁、限流、幂等存储
- 保持现有 API 不变

文档：
- docs/distributed-capabilities.md
- docs/microservices-roadmap.md
- docs/cache.md
- docs/rate-limit.md
- docs/idempotency.md
- docs/observability.md

验收：
1. Java/Koa 都能启动
2. 登录不受影响
3. 关键接口返回结构不变
4. Redis 中能看到限流/幂等/锁相关 key
5. 日志中能看到 requestId/traceId
6. /api/metrics 有新增指标
7. 文档说明当前不是微服务，而是模块化单体 + 分布式能力预留
