# 多租户隔离改造要求

请重构当前 Koa + TypeScript 后端的多租户隔离流程，确保所有租户级数据都严格按当前登录用户的 `tenantId` 隔离。

---

## 1. 总体目标

系统是多租户 SaaS 后端。登录用户的上下文中包含类似如下信息：

```json
{
  "userId": "100001",
  "username": "admin",
  "tenantId": "100000",
  "isAdmin": "1",
  "roles": ["tenant_admin"]
}
```

当该用户访问角色列表、用户列表、部门列表等租户级接口时，SQL 必须自动追加：

```sql
tenant_id = '100000'
```

不能查到：

```sql
tenant_id = '000000'
```

的数据。

---

## 2. 核心原则

1. 超管也必须租户隔离。
2. 不允许因为 `tenantId = 000000` 就跳过租户隔离。
3. `000000` 仅表示平台租户 ID，不代表可以查全库。
4. 业务 Repository 不要手写 `tenant_id = :tenantId`。
5. 租户条件必须统一由 `tenantWhere()` 和 `BaseCrudRepository` 自动拼接。
6. 登录接口、验证码接口、健康检查接口可以没有 `tenantId`。
7. 但只要访问租户级表，必须有当前 `tenantId`，否则直接报错。
8. `sys_menu`、`sys_package`、`sys_package_menu` 是平台全局表，不走租户隔离。
9. `sys_role`、`sys_user`、`sys_dept`、`sys_post`、`sys_config`、`sys_dict`、`sys_theme` 等租户业务表必须走租户隔离。
10. `sys_tenant` 属于平台管理表，是否隔离由具体接口通过 `tenantScoped: false` 控制，不要默认混入普通业务查询。

## 3. 改造要求

### 3.1 tenant 常量文件

请检查并修正为：

```ts
export const PLATFORM_TENANT_ID = "000000";
export const TENANT_ID_FIELD = "tenant_id";

export function isPlatformTenantId(
  tenantId: string | number | null | undefined,
): boolean {
  return String(tenantId ?? "") === PLATFORM_TENANT_ID;
}

export function normalizeTenantId(
  tenantId: string | number | null | undefined,
): string {
  if (tenantId === null || tenantId === undefined || tenantId === "") {
    throw new Error("缺少 tenantId");
  }

  return String(tenantId);
}
```

注意：不要写成下面这样，因为会把缺失的 `tenantId` 自动变成 `000000`，导致租户隔离失效。

```ts
return String(tenantId ?? PLATFORM_TENANT_ID);
```

### 3.2 使用 AsyncLocalStorage 保存当前请求租户

实现或修正如下结构：

```ts
import { AsyncLocalStorage } from "node:async_hooks";

export interface TenantContext {
  tenantId: string | null;
}

export const TenantStorage = new AsyncLocalStorage<TenantContext>();

export function getCurrentTenantId(): string | null {
  return TenantStorage.getStore()?.tenantId ?? null;
}
```

要求：

- 每个请求必须有独立的租户上下文。
- 不能用全局变量保存 `tenantId`。

### 3.3 tenantMiddleware 改造

当前代码应改成更安全的版本：

```ts
export async function tenantMiddleware(
  ctx: Context,
  next: Next,
): Promise<void> {
  const rawTenantId = ctx.state.user?.tenantId;

  const tenantId =
    rawTenantId === null || rawTenantId === undefined || rawTenantId === ""
      ? null
      : normalizeTenantId(rawTenantId);

  await TenantStorage.run({ tenantId }, async () => {
    await next();
  });
}
```

要求：

1. `tenantMiddleware` 不要直接抛“缺少 tenantId”。
2. 未登录接口允许 `tenantId` 为 `null`。
3. 真正访问租户级表时，由 `tenantWhere()` 抛错。
4. `tenantMiddleware` 必须放在 `authMiddleware` 后面，因为它依赖 `ctx.state.user`。

中间件顺序应类似：

```ts
app.use(bodyParser());
app.use(authMiddleware);
app.use(tenantMiddleware);
app.use(router.routes());
app.use(router.allowedMethods());
```

如果当前项目的 `authMiddleware` 会对登录接口跳过处理，也没关系，`tenantMiddleware` 对登录接口保存 `tenantId: null` 即可。

### 3.4 tenantWhere 改造

实现或修正如下：

```ts
export function tenantWhere(
  table: string,
  alias?: string,
): { sql: string; params: Record<string, unknown> } {
  const globalTables = ["sys_menu", "sys_package", "sys_package_menu"];

  if (globalTables.includes(table)) {
    return { sql: "", params: {} };
  }

  const tenantId = getCurrentTenantId();

  if (!tenantId) {
    throw new Error("缺少 tenantId，禁止访问租户数据");
  }

  const column = alias ? `${alias}.${TENANT_ID_FIELD}` : TENANT_ID_FIELD;

  return {
    sql: `${column} = :tenantId`,
    params: {
      tenantId,
    },
  };
}
```

重点：

- 不要加下面这段逻辑：

```ts
if (isPlatformTenantId(getCurrentTenantId())) {
  return { sql: "", params: {} };
}
```

因为当前系统要求：超管也要租户隔离。

### 3.5 修正 BaseCrudRepository.list

请确保 `BaseCrudRepository.list` 统一拼接租户条件、关键词、状态、额外条件。

逻辑应类似：

```ts
export interface BaseListOptions {
  table: string;
  alias?: string;
  idColumn: string;
  selectSql: string;
  orderBy?: string;
  tenantScoped?: boolean;
  keywordColumn?: string;
  keyword?: unknown;
  status?: unknown;
  extraConditions?: SqlFragment[];
}

export class BaseCrudRepository {
  async list<T>(
    options: BaseListOptions,
    page: PageParams,
  ): Promise<{ rows: T[]; total: number }> {
    const tenant =
      options.tenantScoped === false
        ? { sql: "", params: {} }
        : tenantWhere(options.table, options.alias);

    const where = joinConditions([
      tenant,
      options.keywordColumn
        ? likeCondition(options.keywordColumn, "keyword", options.keyword)
        : { sql: "", params: {} },
      eqCondition(
        options.alias ? `${options.alias}.status` : "status",
        "status",
        options.status,
      ),
      ...(options.extraConditions ?? []),
    ]);

    const orderBy = options.orderBy ?? `${options.idColumn} DESC`;

    const params = {
      ...where.params,
      offset: page.offset,
      pageSize: page.pageSize,
    };

    const rowsSql = `
      ${options.selectSql}
      ${where.sql}
      ORDER BY ${orderBy}
      LIMIT :offset, :pageSize
    `;

    const fromTable = options.alias
      ? `${options.table} ${options.alias}`
      : options.table;

    const totalSql = `
      SELECT COUNT(*) AS total
      FROM ${fromTable}
      ${where.sql}
    `;

    console.log("[BaseCrudRepository.list] table:", options.table);
    console.log("[BaseCrudRepository.list] tenantScoped:", options.tenantScoped);
    console.log("[BaseCrudRepository.list] currentTenantId:", getCurrentTenantId());
    console.log("[BaseCrudRepository.list] where.sql:", where.sql);
    console.log("[BaseCrudRepository.list] where.params:", where.params);
    console.log("[BaseCrudRepository.list] rowsSql:", rowsSql);
    console.log("[BaseCrudRepository.list] totalSql:", totalSql);

    const rows = await query<T>(rowsSql, params);

    const total = await queryOne<{ total: number }>(totalSql, where.params);

    return {
      rows,
      total: total?.total ?? 0,
    };
  }
}
```

注意：

- 如果使用 alias，例如 `FROM sys_role r`，则 `alias: "r"`。
- `tenantWhere` 应生成：

```sql
r.tenant_id = :tenantId
```

- `totalSql` 也必须是：

```sql
FROM sys_role r
```

否则会报 alias 找不到。

### 3.6 修正 joinConditions

请检查 `joinConditions()`，必须保证它能把多个条件正确拼成 `WHERE ... AND ...`。

建议实现如下：

```ts
export interface SqlFragment {
  sql: string;
  params: Record<string, unknown>;
}

export function joinConditions(conditions: SqlFragment[]): SqlFragment {
  const validConditions = conditions.filter((condition) => {
    return condition.sql && condition.sql.trim().length > 0;
  });

  if (validConditions.length === 0) {
    return {
      sql: "",
      params: {},
    };
  }

  return {
    sql: `WHERE ${validConditions.map((item) => item.sql).join(" AND ")}`,
    params: Object.assign({}, ...validConditions.map((item) => item.params)),
  };
}
```

新增角色时，`tenantId` 必须从当前请求上下文获取，不能由前端传入，也不能默认写 `000000`。

async createRole(dto: CreateRoleDto) {

    roleName: dto.roleName,

要求：

1. 不允许使用 `dto.tenantId`。
2. 不允许默认写 `PLATFORM_TENANT_ID`。
3. 当前登录用户 `tenantId = 100000` 时，新增角色必须写入 `tenant_id = 100000`。

### 3.9 修正 update 和 softDelete

更新和删除必须自动带租户条件。

示例：

```ts
update(
  table: string,
  idColumn: string,
  id: string,
  data: Record<string, unknown>,
  tenantScoped = true,
): Promise<unknown> {
  const keys = Object.keys(data);
  const sets = keys.map(
    (key) =>
      `${key.replace(/[A-Z]/g, (char) => `_${char.toLowerCase()}`)} = :${key}`,
  );

  const tenant = tenantScoped ? tenantWhere(table) : { sql: "", params: {} };
  const tenantSql = tenant.sql ? ` AND ${tenant.sql}` : "";

  return execute(
    `UPDATE ${table} SET ${sets.join(", ")} WHERE ${idColumn} = :id${tenantSql}`,
    { ...data, id, ...tenant.params },
  );
}
```

删除示例：

```ts
softDelete(
  table: string,
  idColumn: string,
  ids: string[],
  tenantScoped = true,
): Promise<unknown> {
  const tenant = tenantScoped ? tenantWhere(table) : { sql: "", params: {} };

  const idParams = Object.fromEntries(
    ids.map((id, index) => [`id${index}`, id]),
  );

  const idPlaceholders = ids.map((_, index) => `:id${index}`).join(", ");

  const tenantSql = tenant.sql ? ` AND ${tenant.sql}` : "";

  return execute(
    `UPDATE ${table} SET deleted = 1 WHERE ${idColumn} IN (${idPlaceholders})${tenantSql}`,
    { ...idParams, ...tenant.params },
  );
}
```

这样普通租户无法通过传入其他租户的 ID 修改或删除数据。

### 3.11 排查为什么登录 tenantId 是 100000，却查到了 000000

请重点排查以下位置：

#### A. `listRoles` 有没有传 `tenantScoped: false`

如果有，删除。

```ts
tenantScoped: false
```

#### B. `sys_role` 是否被放进了 `globalTables`

如果有，删除。

#### C. `listRoles` 的 `selectSql` 是否还有手写 `WHERE`

如果有，删除：

```sql
WHERE tenant_id = :tenantId
```

#### D. `tenantMiddleware` 是否在 `authMiddleware` 后面

必须保证：

```ts
app.use(authMiddleware);
app.use(tenantMiddleware);
```

#### E. `ctx.state.user.tenantId` 是否真的等于 `100000`

在 `tenantMiddleware` 里临时打印：

```ts
console.log("[tenantMiddleware] user:", ctx.state.user);
console.log("[tenantMiddleware] tenantId:", tenantId);
```

#### F. `BaseCrudRepository.list` 最终 SQL 是否真的有租户条件

必须看到类似日志：

```txt
currentTenantId: 100000
where.sql: WHERE tenant_id = :tenantId AND deleted = 0
where.params: { tenantId: '100000' }
```

如果没有，说明 `tenantWhere()` 没有接入成功。

## 4. 正确期望结果

当前登录用户：

```json
{
  "tenantId": "100000"
}
```

访问角色列表时，最终 SQL 应该类似：

```sql
SELECT 
  role_id AS roleId,
  tenant_id AS tenantId,
  role_name AS roleName,
  role_key AS roleKey,
  sort_num AS sortNum,
  status,
  remark,
  create_time AS createTime,
  update_time AS updateTime
FROM sys_role
WHERE tenant_id = :tenantId
AND deleted = 0
ORDER BY sort_num ASC, role_id DESC
LIMIT :offset, :pageSize
```

参数应该是：

```ts
{
  tenantId: "100000",
  offset: 0,
  pageSize: 10
}
```

返回结果中不应该出现：

```json
"tenantId": "000000"
```

如果返回了，说明最终执行的 SQL 没有带上 `tenant_id = '100000'`。
