# BLS-Server Java 后端迁移设计规范

> 目标：将 bls-server 从 Koa/TypeScript/Kysely 迁移到 Spring Boot 3.x/Java 17/MyBatis-Plus，前端零改动。

## 1. 技术选型

| 层次 | 原（Node.js） | 目标（Java） |
|------|-------------|------------|
| 框架 | Koa 2.x | Spring Boot 3.x |
| 语言 | TypeScript | Java 17+ |
| ORM | Kysely | MyBatis-Plus 3.5+ |
| 缓存 | ioredis | Spring Data Redis (Lettuce) |
| 认证 | jsonwebtoken | jjwt |
| 校验 | zod | Jakarta Validation |
| 文件 | MinIO SDK | MinIO Java SDK |
| Excel | exceljs | EasyExcel |

## 2. 包结构

```
com.bls
├── common/          # 公共：config/context/exception/handler/response/utils
├── module/
│   ├── auth/        # 登录/登出/刷新
│   └── system/      # user/dept/role/menu/tenant/package/config/dict/theme/storage/file/log/global-search
└── infrastructure/  # 多租户插件、存储抽象层
```

## 3. 关键设计

### 统一响应 `R<T>`

```java
// 成功：{ code: 200, message: "ok", data: {...} }
// 分页：{ code: 200, data: [...], total: 100 }
// 错误：{ code: 500, message: "..." }
// Session失效：{ code: 40101, message: "会话已失效" }  // 前端区分普通401
```

### JWT 双 Token

- Access Token：15分钟，`Bearer` 前缀，Payload 含 `{userId, username, tenantId, jti, tokenType:"access"}`
- Refresh Token：7天，无前缀，一次性使用（复用检测→吊销所有会话）
- Redis Session：key = `session:{jti}`

### 多租户

- ThreadLocal `TenantContext` + MyBatis-Plus `TenantLineHandler` 自动拼接 `tenant_id`
- 平台租户 ID = `"000000"`（穿透隔离、拥有全部权限）
- 全局表（`sys_menu`、`sys_package`、`sys_package_menu`）不隔离
- 域名绑定登录：`sys_tenant.domain_name` → 匹配请求 Host

### 通用 CRUD 规范

| 操作 | HTTP | URL 模式 |
|------|------|----------|
| 列表 | GET | `{basePath}/list` |
| 新增 | POST | `{basePath}/add` |
| 编辑 | PUT | `{basePath}/edit` |
| 删除 | DELETE | `{basePath}/remove?ids=id1,id2` |
| 状态 | PUT | `{basePath}/status` |

- 分页参数：`pageNum`/`pageSize`
- 状态字段：`status = '0'`（正常）/ `'1'`（停用），`deleted = 0/1`（逻辑删除）

### 存储抽象

`StorageService` 接口（upload/getUrl/download/delete），多实现（MinIO/OSS/COS/Local），通过 `sys_storage_config` 表动态选择。

### 错误码

| code | HTTP | 含义 |
|------|------|------|
| 200 | 200 | 成功 |
| 400 | 400 | 参数错误 |
| 401 | 401 | 未登录/Token过期 |
| 40101 | 401 | 会话失效（被踢下线） |
| 403 | 403 | 无权限 |
| 404 | 404 | 资源不存在 |
| 500 | 500 | 服务器错误 |

## 4. API 接口清单

### 认证 `/api/auth`
| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/login` | 无 | 登录（支持 tenantId/域名识别） |
| POST | `/refresh` | 无 | 刷新 Token |
| GET | `/profile` | JWT | 当前用户信息 |
| POST | `/logout` | JWT(可选) | 登出 |

### 系统管理 `/api/system/*`
按模块：dept / menu / role / user / tenant / package / config / dict / theme / storage / file / log / realtime / global-search / page-config，每个模块标准 CRUD + 特殊端点。

### 内容管理 `/api/content/*`
news / meeting / special-guest / download，每个模块标准 CRUD。

### Excel `/api/common/excel`
template / export / import（推荐 EasyExcel）。

## 5. 迁移检查清单

- [ ] JWT 双 Token + Refresh 旋转，前端 401 自动刷新
- [ ] Redis Session 结构保持 `session:` 前缀
- [ ] 分页参数 `pageNum`/`pageSize` 从 query string 读取
- [ ] 删除支持 `?ids=` 和 `{ids:[]}` 两种方式
- [ ] 平台租户 `000000` 穿透隔离
- [ ] 菜单/套餐为全局表，不隔离
- [ ] 密码 MD5 保持兼容
- [ ] 文件上传返回格式与前端 `FileUpload` 组件对齐
- [ ] 操作日志 AOP 自动记录
- [ ] WebSocket realtime 保持 JWT 认证 + 3s 广播 + 15s 心跳
