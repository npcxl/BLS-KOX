## 2. 统一响应格式

### 2.1 响应体结构

前端期望的响应结构（来自 `requestErrorConfig.ts` 和所有 API 调用）:

```java
// 成功响应
{
  "code": 200,
  "message": "ok",
  "data": { ... }
}

// 分页数据响应
{
  "code": 200,
  "message": "ok",
  "data": [ ... ],
  "total": 100
}

// 错误响应
{
  "code": 500,
  "message": "服务器内部错误"
}

// 401 - session 失效特殊错误码
{
  "code": 40101,
  "message": "会话已失效，请重新登录"
}
```

### 2.2 Java 实现

```java
package com.bls.common.response;

import lombok.Data;
import java.io.Serializable;

@Data
public class R<T> implements Serializable {
    private int code;
    private String message;
    private T data;
    private Long total;  // 分页总记录数（仅分页接口使用）

    private R() {}

    public static <T> R<T> ok(T data) {
        R<T> r = new R<>();
        r.code = 200;
        r.message = "ok";
        r.data = data;
        return r;
    }

    public static <T> R<T> ok(T data, Long total) {
        R<T> r = ok(data);
        r.total = total;
        return r;
    }

    public static <T> R<T> ok(String message, T data) {
        R<T> r = ok(data);
        r.message = message;
        return r;
    }

    public static <T> R<T> fail(int code, String message) {
        R<T> r = new R<>();
        r.code = code;
        r.message = message;
        return r;
    }

    public static <T> R<T> fail(String message) {
        return fail(500, message);
    }
}
```

> **关键点**：所有的 Controller 方法返回值必须为 `R<T>`，确保前端不修改任何错误处理逻辑。

## 3. 认证与授权

### 3.1 认证流程

```
┌──────────┐     POST /api/auth/login       ┌──────────┐
│  前端     │ ──────────────────────────────>  │  Java    │
│ (React)  │     { username, password,        │  后端    │
│          │       tenantId? }                │          │
│          │ <────────────────────────────── │          │
│          │     { token, refreshToken,       │          │
│          │       code:200, data:{...} }     │          │
└──────────┘                                  └──────────┘
     │                                              │
     │  后续请求 Header: Authorization: Bearer {token}
     │──────────────────────────────────────────────>
     │                                              │
     │  401 时自动 POST /api/auth/refresh           │
     │──────────────────────────────────────────────>
     │  { refreshToken }                            │
     │ <──────────────────────────────────────────────
     │  { token, refreshToken }
```

### 3.2 JWT Token 规范

| 项目 | 值 |
|------|-----|
| **签发方式** | `Bearer {token}` |
| **Access Token 有效期** | 15 分钟 |
| **Refresh Token 有效期** | 7 天 |
| **签名算法** | HS256 |
| **Payload 字段** | `userId`, `username`, `tenantId`, `jti` (UUID), `tokenType` (access/refresh) |
| **存储** | Redis，key = `session:{jti}` |

> **重要**: 前端登录时将密码做 MD5 后发送，后端直接比对数据库中的 MD5 值。

```java
package com.bls.common.utils;

import io.jsonwebtoken.*;
import io.jsonwebtoken.security.Keys;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

import javax.crypto.SecretKey;
import java.nio.charset.StandardCharsets;
import java.util.Date;
import java.util.UUID;

@Component
public class JwtUtils {

    @Value("${bls.jwt.secret}")
    private String secret;

    @Value("${bls.jwt.expires-in}")
    private long expiresIn;          // 900 (15分钟)

    @Value("${bls.jwt.refresh-expires-in}")
    private long refreshExpiresIn;   // 604800 (7天)

    private SecretKey getKey() {
        return Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
    }

    /**
     * 签发 Access Token，返回格式: "Bearer eyJhbGciOi..."
     * 前端直接取用，无需再拼接 Bearer
     */
    public String signToken(String userId, String username, String tenantId) {
        String jti = UUID.randomUUID().toString();
        Date now = new Date();

        String token = Jwts.builder()
                .claim("userId", userId)
                .claim("username", username)
                .claim("tenantId", tenantId)
                .claim("jti", jti)
                .claim("tokenType", "access")
                .setIssuedAt(now)
                .setExpiration(new Date(now.getTime() + expiresIn * 1000))
                .signWith(getKey())
                .compact();

        return "Bearer " + token;
    }

    /**
     * 签发 Refresh Token（不含 Bearer 前缀）
     */
    public String signRefreshToken(String userId, String username, String tenantId) {
        String jti = UUID.randomUUID().toString();
        Date now = new Date();

        return Jwts.builder()
                .claim("userId", userId)
                .claim("username", username)
                .claim("tenantId", tenantId)
                .claim("jti", jti)
                .claim("tokenType", "refresh")
                .setIssuedAt(now)
                .setExpiration(new Date(now.getTime() + refreshExpiresIn * 1000))
                .signWith(getKey())
                .compact();
    }

    /**
     * 验证 Access Token 并返回 Claims
     */
    public Claims verifyToken(String token) {
        Claims claims = Jwts.parserBuilder()
                .setSigningKey(getKey())
                .build()
                .parseClaimsJws(token)
                .getBody();

        if (!"access".equals(claims.get("tokenType"))) {
            throw new JwtException("invalid access token");
        }
        return claims;
    }

    /**
     * 验证 Refresh Token 并返回 Claims
     */
    public Claims verifyRefreshToken(String token) {
        Claims claims = Jwts.parserBuilder()
                .setSigningKey(getKey())
                .build()
                .parseClaimsJws(token)
                .getBody();

        if (!"refresh".equals(claims.get("tokenType"))) {
            throw new JwtException("invalid refresh token");
        }
        return claims;
    }

    /**
     * 从 Authorization Header 中提取纯 token
     * "Bearer xxx" -> "xxx"
     */
    public static String parseBearerToken(String authorization) {
        if (authorization == null || !authorization.startsWith("Bearer ")) {
            return null;
        }
        return authorization.substring(7);
    }
}
```

### 3.3 Redis Session 管理

```java
package com.bls.module.auth.service;

import com.bls.common.utils.JwtUtils;
import io.jsonwebtoken.Claims;
import lombok.RequiredArgsConstructor;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Service;

import java.util.concurrent.TimeUnit;

@Service
@RequiredArgsConstructor
public class SessionService {

    private final RedisTemplate<String, Object> redisTemplate;
    private final JwtUtils jwtUtils;

    private static final String SESSION_PREFIX = "session:";

    /**
     * 保存 session，过期时间与 access token 一致
     */
    public void saveSession(Claims claims) {
        String jti = claims.get("jti", String.class);
        SessionInfo info = new SessionInfo();
        info.setUserId(claims.get("userId", String.class));
        info.setUsername(claims.get("username", String.class));
        info.setTenantId(claims.get("tenantId", String.class));
        // 15 分钟后过期
        redisTemplate.opsForValue().set(SESSION_PREFIX + jti, info, 15, TimeUnit.MINUTES);
    }

    /**
     * 获取并刷新 session（每次请求时刷新 TTL）
     */
    public SessionInfo getSession(String jti) {
        Object value = redisTemplate.opsForValue().get(SESSION_PREFIX + jti);
        if (value instanceof SessionInfo info) {
            // 刷新过期时间
            redisTemplate.expire(SESSION_PREFIX + jti, 15, TimeUnit.MINUTES);
            return info;
        }
        return null;
    }

    /**
     * 删除 session（登出时调用）
     */
    public void removeSession(String jti) {
        redisTemplate.delete(SESSION_PREFIX + jti);
    }
}
```

### 3.4 密码工具

```java
package com.bls.common.utils;

import org.apache.commons.codec.digest.DigestUtils;

public class PasswordUtils {

    /**
     * MD5 哈希
     * 前端已做 MD5，后端直接比对 hash
     */
    public static String hash(String password) {
        return DigestUtils.md5Hex(password);
    }

    /**
     * 验证密码
     * password: 前端传来的值（已 MD5 或明文）
     * hash: 数据库存储的 MD5
     */
    public static boolean verify(String password, String hash) {
        // 如果前端传了 32 位 MD5，直接比对
        String inputHash = password.length() == 32
                ? password.toLowerCase()
                : DigestUtils.md5Hex(password).toLowerCase();
        return inputHash.equals(hash.toLowerCase());
    }
}
```

### 3.5 JWT 认证拦截器

```java
package com.bls.common.interceptor;

import com.bls.common.context.UserContext;
import com.bls.common.exception.SessionInvalidException;
import com.bls.common.exception.UnauthorizedException;
import com.bls.common.utils.JwtUtils;
import com.bls.module.auth.service.SessionService;
import io.jsonwebtoken.Claims;
import io.jsonwebtoken.ExpiredJwtException;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Component;
import org.springframework.web.method.HandlerMethod;
import org.springframework.web.servlet.HandlerInterceptor;

@Component
@RequiredArgsConstructor
public class JwtAuthInterceptor implements HandlerInterceptor {

    private final JwtUtils jwtUtils;
    private final SessionService sessionService;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) throws Exception {
        // 检查是否标注了 @JwtAuth 注解
        if (!(handler instanceof HandlerMethod hm)) return true;

        JwtAuth jwtAuth = hm.getMethodAnnotation(JwtAuth.class);
        if (jwtAuth == null) {
            jwtAuth = hm.getBeanType().getAnnotation(JwtAuth.class);
        }
        if (jwtAuth == null) return true;  // 无需认证

        String rawToken = JwtUtils.parseBearerToken(request.getHeader("Authorization"));
        if (rawToken == null) {
            if (jwtAuth.optional()) return true;
            throw new UnauthorizedException();
        }

        try {
            Claims claims = jwtUtils.verifyToken(rawToken);
            SessionService.SessionInfo session = sessionService.getSession(
                    claims.get("jti", String.class));

            if (session == null ||
                !session.getUserId().equals(claims.get("userId", String.class))) {
                throw new SessionInvalidException();
            }

            // 设置当前用户上下文
            UserContext.set(claims);
            return true;

        } catch (ExpiredJwtException e) {
            throw new UnauthorizedException("登录已过期");
        } catch (UnauthorizedException | SessionInvalidException e) {
            throw e;
        } catch (Exception e) {
            throw new UnauthorizedException();
        }
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                 Object handler, Exception ex) {
        UserContext.clear();
    }
}

// 注解定义
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface JwtAuth {
    boolean optional() default false;
}
```

### 3.6 权限检查注解/拦截器

```java
package com.bls.common.interceptor;

// 权限注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface HasPerm {
    String value();  // 如 "system:user:list"
}

// 权限拦截器（在 JwtAuthInterceptor 之后执行）
@Component
@RequiredArgsConstructor
public class PermissionInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) throws Exception {
        if (!(handler instanceof HandlerMethod hm)) return true;

        HasPerm hasPerm = hm.getMethodAnnotation(HasPerm.class);
        if (hasPerm == null) return true;

        // 平台租户 "000000" 拥有全部权限
        String tenantId = UserContext.getTenantId();
        if ("000000".equals(tenantId)) return true;

        // 检查用户权限列表
        String requiredPerm = hasPerm.value();
        if (UserContext.hasPermission(requiredPerm) || UserContext.hasPermission("*")) {
            return true;
        }

        throw new ForbiddenException();
    }
}
```

## 4. 多租户设计

### 4.1 租户上下文

使用 **ThreadLocal** 存储当前请求的租户 ID，通过拦截器自动解析。

```java
package com.bls.common.context;

import com.bls.common.dto.CurrentUser;

public class TenantContext {
    private static final ThreadLocal<String> TENANT_HOLDER = new ThreadLocal<>();
    private static final String PLATFORM_TENANT = "000000";

    public static void set(String tenantId) {
        TENANT_HOLDER.set(tenantId);
    }

    public static String get() {
        String id = TENANT_HOLDER.get();
        if (id == null || id.isEmpty()) {
            throw new AppException("缺少 tenantId，禁止访问租户数据");
        }
        return id;
    }

    public static boolean isPlatform() {
        return PLATFORM_TENANT.equals(TENANT_HOLDER.get());
    }

    public static void clear() {
        TENANT_HOLDER.remove();
    }
}
```

### 4.2 MyBatis-Plus 多租户插件

使用 MyBatis-Plus 的 `TenantLineHandler` 自动为 SQL 拼接 `tenant_id` 条件。

```java
package com.bls.infrastructure.tenant;

import com.baomidou.mybatisplus.extension.plugins.handler.TenantLineHandler;
import com.bls.common.context.TenantContext;
import net.sf.jsqlparser.expression.Expression;
import net.sf.jsqlparser.expression.StringValue;

import java.util.Set;

public class BlsTenantLineHandler implements TenantLineHandler {

    // 全局表（不隔离）：sys_menu, sys_package, sys_package_menu
    private static final Set<String> GLOBAL_TABLES = Set.of(
            "sys_menu", "sys_package", "sys_package_menu"
    );

    @Override
    public Expression getTenantId() {
        return new StringValue(TenantContext.get());
    }

    @Override
    public String getTenantIdColumn() {
        return "tenant_id";
    }

    @Override
    public boolean ignoreTable(String tableName) {
        return GLOBAL_TABLES.contains(tableName)
                || TenantContext.isPlatform();  // 平台租户不限制
    }
}
```

public R<List<ProductVO>> list(@RequestParam(required = false) Map<String, String> params) {

## 9. WebSocket 实时推送

### 9.1 协议

```
连接端点: ws://host:port/ws/realtime (可配置)
消息类型: JSON

// 客户端认证
→ { "type": "auth", "token": "eyJ..." }
← { "type": "realtime-info", "data": { cpuUsagePercent: 45.2, ... } }

// 服务端广播 (每 3 秒)
← { "type": "realtime-info", "data": { ... } }

// 心跳 (每 15 秒, 服务端发 ping)
→ pong (WebSocket 协议层)
```

### 9.2 Java 实现要点

```java
@Configuration
@EnableWebSocket
public class WebSocketConfig implements WebSocketConfigurer {

    @Override
    public void registerWebSocketHandlers(WebSocketHandlerRegistry registry) {
        registry.addHandler(realtimeWebSocketHandler(), "/ws/realtime")
                .setAllowedOrigins("*");
    }
}
```

- 认证：客户端发送 `auth` 消息，验证 JWT token
- 广播定时器：3 秒间隔向所有已认证客户端推送系统信息
- 心跳：15 秒间隔检测连接活性，未响应 ping 则断开
- 平台无关：不限租户，所有客户端看到相同的系统信息

## 14. 错误处理

### 14.1 全局异常处理器

```java
package com.bls.common.handler;

import com.bls.common.exception.*;
import com.bls.common.response.R;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.HttpStatus;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ResponseStatus;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(UnauthorizedException.class)
    @ResponseStatus(HttpStatus.UNAUTHORIZED)
    public R<Void> handleUnauthorized(UnauthorizedException e) {
        return R.fail(401, e.getMessage());
    }

    @ExceptionHandler(SessionInvalidException.class)
    @ResponseStatus(HttpStatus.UNAUTHORIZED)
    public R<Void> handleSessionInvalid(SessionInvalidException e) {
        return R.fail(40101, e.getMessage());  // 前端区分普通401和session失效
    }

    @ExceptionHandler(ForbiddenException.class)
    @ResponseStatus(HttpStatus.FORBIDDEN)
    public R<Void> handleForbidden(ForbiddenException e) {
        return R.fail(403, e.getMessage());
    }

    @ExceptionHandler(NotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public R<Void> handleNotFound(NotFoundException e) {
        return R.fail(404, e.getMessage());
    }

    @ExceptionHandler(ValidationException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public R<Void> handleValidation(ValidationException e) {
        return R.fail(400, e.getMessage());
    }

    @ExceptionHandler(AppException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public R<Void> handleAppException(AppException e) {
        return R.fail(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public R<Void> handleException(Exception e) {
        log.error("未捕获异常", e);
        return R.fail(500, "服务器内部错误");
    }
}
```

### 14.2 前端请求拦截器适配

前端 `requestErrorConfig.ts` 处理逻辑：

1. **401 处理**:
   - 普通 401 (`code=401`): 尝试刷新 token → 重试请求
   - Session 失效 (`code=40101`): 弹窗提示"该账号在别处登录"
   - 刷新失败: 跳转登录页

2. **错误信息提取**: `response.data.message`

**后端必须保证**:
- 所有错误响应的 HTTP 状态码与 `code` 字段一致
- 401 返回 `{ code: 401 }`，40101 返回 `{ code: 40101 }`
- `message` 字段始终存在

package com.bls.common.config;

import org.springframework.context.annotation.Configuration;

## 17. 迁移检查清单

### 18.1 前置条件

- [ ] MySQL 数据库保持原有 schema 不变
- [ ] Redis 保持原有数据结构不变 (session 前缀 `session:`)
- [ ] MinIO / OSS 等对象存储配置保持不变
- [ ] 前端无需任何代码修改

### 18.2 接口兼容性验证清单

按模块逐项验证：

- [ ] **Auth**: login / refresh / profile / logout — JWT 双 token 机制
- [ ] **Dept**: list / add / edit / remove — 树形部门
- [ ] **Menu**: list / package-tree / add / edit / remove — 全局表
- [ ] **Role**: list / add / edit / remove — 关联菜单
- [ ] **User**: list / profile / add / edit / password / remove / status
- [ ] **Tenant**: public-list / list / add / edit / remove
- [ ] **Package**: list / options / detail / add / edit / remove
- [ ] **Config**: public-theme / public-system / current / list / add / edit / remove
- [ ] **Dict**: type/list / type/add / type/edit / type/remove / data/type / data/add / data/edit / data/remove
- [ ] **Theme**: current / list / add / edit
- [ ] **Storage**: config/list / config/add / config/edit / config/remove
- [ ] **File**: upload / list / remove / download
- [ ] **Log**: login / operation / upload — 分页查询
- [ ] **GlobalSearch**: search / config CRUD / index rebuild
- [ ] **Content** × 4: CRUD (list / detail / add / edit / remove × 4 模块)
- [ ] **Excel**: template / export / import
- [ ] **Realtime**: HTTP info / WebSocket 广播 + 认证
- [ ] **Health**: /api/health 健康检查

### 18.3 关键风险点

| 风险 | 说明 | 应对 |
|------|------|------|
| **密码 MD5** | 前端已 MD5，后端直接比对 | 保持MD5不变，新建用户也用MD5 |
| **双 Token 刷新** | 前端 401 时自动刷新 | 必须支持 refresh token 旋转 |
| **Session 存储** | Redis key 前缀 `session:` | 数据结构保持一致 |
| **分页参数** | `pageNum` / `pageSize` 查询参数 | 必须从 query string 读取 |
| **删除 ids 参数** | `?ids=id1,id2` 或 `{ids:[...]}` | 两种方式都支持 |
| **菜单全表** | sys_menu, sys_package 不隔离 | 租户插件 ignoreTable 配置 |
| **平台租户** | tenantId="000000" 特殊处理 | 跳过权限检查，看全部数据 |
| **文件路径** | 上传返回的 URL 格式 | 与前端 FileUpload 组件兼容 |
