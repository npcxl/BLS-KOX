# 多租户全局搜索执行方案

## 目标

实现一套适合当前系统的全局搜索能力：

* 使用 MySQL 统一搜索索引表，避免每次搜索扫描多张业务表。
* 使用 Redis 缓存用户权限、角色、搜索配置，减少权限查询压力。
* 支持多租户隔离。
* 支持普通用户按权限搜索。
* 支持超管前端配置搜索模块。
* 超管拥有功能配置权限，但搜索结果仍然必须遵守当前租户上下文隔离。
* 全局搜索只返回摘要信息，不返回敏感字段。

---

# 一、核心原则

## 1. 全局搜索不直接查业务表

不要每次全局搜索都动态搜索：

```text
sys_user
sys_role
biz_exhibitor
biz_order
biz_file
...
```

而是提前把可搜索内容同步到统一索引表：

```text
sys_search_index
```

搜索时只查这一张表。

---

## 2. 权限控制必须动态化

用户能搜到什么，由以下因素决定：

```text
tenant_id 当前租户
user_id 当前用户
role_id 当前角色
permission 当前权限
data_scope 数据权限
module_key 搜索模块
```

最终规则：

```text
用户属于哪个租户，只能搜哪个租户的数据；
用户拥有哪些角色，决定有哪些权限；
用户拥有哪些权限，决定能搜哪些模块；
角色的数据范围，决定模块内能搜到哪些数据；
超管也不能默认跨租户搜索，必须基于当前租户上下文。
```

# 二、数据库设计

## 1. 新增统一搜索索引表

创建表：

```sql
CREATE TABLE IF NOT EXISTS sys_search_index (
  index_id varchar(32) NOT NULL COMMENT '索引ID',

  tenant_id varchar(32) NOT NULL COMMENT '租户ID',
  module_key varchar(64) NOT NULL COMMENT '模块标识',
  module_name varchar(64) NOT NULL COMMENT '模块名称',

  biz_id varchar(64) NOT NULL COMMENT '业务主键ID',
  title varchar(255) NOT NULL COMMENT '搜索标题',
  subtitle varchar(255) DEFAULT NULL COMMENT '搜索副标题',
  content text DEFAULT NULL COMMENT '搜索内容',

  permission varchar(128) NOT NULL COMMENT '搜索权限标识',
  route_path varchar(255) DEFAULT NULL COMMENT '前端跳转路径',

  owner_id varchar(32) DEFAULT NULL COMMENT '数据所属用户ID',
  dept_id varchar(32) DEFAULT NULL COMMENT '部门ID',
  created_by varchar(32) DEFAULT NULL COMMENT '创建人',

  status varchar(8) DEFAULT '0' COMMENT '状态：0正常 1停用',
  deleted tinyint DEFAULT 0 COMMENT '删除标记：0正常 1删除',

  source_table varchar(128) DEFAULT NULL COMMENT '来源业务表',
  created_at datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

  PRIMARY KEY (index_id),

  KEY idx_search_tenant_module (tenant_id, module_key),
  KEY idx_search_tenant_permission (tenant_id, permission),
  KEY idx_search_tenant_status_deleted (tenant_id, status, deleted),
  KEY idx_search_owner (tenant_id, owner_id),
  KEY idx_search_created_by (tenant_id, created_by),
  KEY idx_search_dept (tenant_id, dept_id),
  KEY idx_search_biz (tenant_id, module_key, biz_id),
  KEY idx_search_title (tenant_id, title),
  KEY idx_search_subtitle (tenant_id, subtitle)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='统一搜索索引表';
```

# 四、权限设计

## 1. 新增搜索权限

建议给菜单权限里补充：

```text
system:user:search
system:role:search
system:menu:search
exhibitor:search
order:search
file:search
```

权限规则：

```text
xxx:list   控制列表页访问
xxx:search 控制全局搜索
xxx:detail 控制详情查看
xxx:create 控制新增
xxx:update 控制修改
xxx:delete 控制删除
xxx:export 控制导出
```

---

## 2. 超管权限

超管可以拥有：

```text
*:*:*
```

但是注意：

```text
*:*:* 只表示功能权限全部拥有；
不代表可以跨租户搜索数据。
```

超管全局搜索时必须带当前租户上下文：

```text
currentTenantId
```

如果当前租户上下文是：

```text
000000
```

则只搜平台数据。

如果当前切换到：

```text
100001
```

则只搜租户 100001 的数据。


```json

  "system:user:search",
  "system:role:search",
  "exhibitor:search"

```



```text

```

# 七、全局搜索后端逻辑

## 1. 搜索主流程

```text
1. 从 ctx.state.user 获取 userId、tenantId、isAdmin。
2. 获取 currentTenantId。
   - 普通用户：currentTenantId = user.tenantId
   - 超管：currentTenantId = 前端传入或当前上下文租户
3. 校验 keyword。
   - keyword 为空，直接返回 []
   - keyword 长度小于 2，直接返回 []
4. 从 Redis 获取用户权限。
5. 从 Redis 获取用户角色。
6. 从 Redis 获取搜索配置。
7. 根据权限过滤配置。
8. 查询 sys_search_index。
9. 追加 tenant_id 隔离。
10. 追加 permission 权限过滤。
11. 追加 data_scope 数据权限过滤。
12. 按 module_key 分组返回。
```

---

## 4. SQL 示例

基础搜索 SQL：

```sql
SELECT
  index_id,
  module_key,
  module_name,
  biz_id AS id,
  title,
  subtitle,
  route_path
FROM sys_search_index
WHERE tenant_id = :tenantId
  AND deleted = 0
  AND status = '0'
  AND permission IN (:permissions)
  AND (
    title LIKE :keyword
    OR subtitle LIKE :keyword
    OR content LIKE :keyword
  )
ORDER BY updated_at DESC
LIMIT :limit;
```

## 5. 数据权限追加

角色 `data_scope` 建议：

```text
1 = 全部数据权限
2 = 本租户数据权限
3 = 本部门数据权限
4 = 本部门及以下数据权限
5 = 仅本人数据权限
```

当前阶段如果没有部门，先实现：

```text
1 = 全部数据权限
2 = 本租户数据权限
5 = 仅本人数据权限
```

注意：

```text
即使 data_scope = 1，也不能跨租户。
data_scope = 1 只代表当前租户内全部数据。
```

数据权限逻辑：

```ts
function buildDataScopeWhere(roles: Role[], userId: string) {
  const hasAll = roles.some(role => role.dataScope === '1');
  if (hasAll) {
    return {
      sql: '',
      params: {},
    };
  }

  const hasTenant = roles.some(role => role.dataScope === '2');
  if (hasTenant) {
    return {
      sql: '',
      params: {},
    };
  }

  const hasSelf = roles.some(role => role.dataScope === '5');
  if (hasSelf) {
    return {
      sql: ' AND (owner_id = :userId OR created_by = :userId) ',
      params: {
        userId,
      },
    };
  }

  return {
    sql: ' AND 1 = 0 ',
    params: {},
  };
}
```

## 2. 删除业务数据时同步删除索引

逻辑删除时：

```sql
UPDATE sys_search_index
SET deleted = 1
WHERE tenant_id = :tenantId
  AND module_key = :moduleKey
  AND biz_id = :bizId;
```

```sql
DELETE FROM sys_search_index
WHERE tenant_id = :tenantId
  AND module_key = :moduleKey
  AND biz_id = :bizId;
```

---

## 3. 重建索引

重建流程：

```text
1. 查询 sys_global_search_config。
2. 如果指定 moduleKey，只处理对应模块。
3. 如果指定 tenantId，只处理对应租户。
4. 先删除旧索引。
5. 根据配置表读取业务表数据。
6. 重新写入 sys_search_index。
```

注意：

```text
动态表名和字段名必须做白名单校验。
不要直接相信数据库配置。
```

字段校验方法：

```ts
function safeIdentifier(value: string) {

    throw new Error(`非法字段或表名：${value}`);
  }

  return value;
```

# 十、后端伪代码

## 1. globalSearch

```ts
export async function globalSearch(ctx: Context) {
  const keyword = String(ctx.query.keyword || '').trim();

  if (!keyword || keyword.length < 2) {
    ctx.body = {
      code: 200,
      message: 'success',
      data: [],
    };
    return;
  }

  const loginUser = ctx.state.user;

  if (!loginUser?.userId) {
    ctx.status = 401;
    ctx.body = {
      code: 401,
      message: '未登录',
    };
    return;
  }

  const userId = loginUser.userId;
  const tenantId = resolveSearchTenantId(ctx, loginUser);

  if (!tenantId) {
    ctx.status = 403;
    ctx.body = {
      code: 403,
      message: '缺少租户上下文，禁止搜索',
    };
    return;
  }

  const [permissions, roles, configs] = await Promise.all([
    getCachedUserPermissions(userId, tenantId),
    getCachedUserRoles(userId, tenantId),
    getCachedGlobalSearchConfigs(),
  ]);

  const allowedPermissions = configs
    .filter(config => hasPermission(permissions, config.permission))
    .filter(config => config.enabled === 1)
    .map(config => config.permission);

  if (allowedPermissions.length === 0) {
    ctx.body = {
      code: 200,
      message: 'success',
      data: [],
    };
    return;
  }

  const list = await searchIndexRepository.search({
    tenantId,
    keyword,
    permissions: allowedPermissions,
    roles,
    userId,
    limit: 30,
  });

  ctx.body = {
    code: 200,
    message: 'success',
    data: groupSearchResults(list),
  };
}
```

## 2. resolveSearchTenantId

```ts
function resolveSearchTenantId(ctx: Context, loginUser: any) {
  const currentTenantId =
    ctx.headers['x-tenant-id'] ||
    ctx.query.tenantId ||
    loginUser.currentTenantId ||
    loginUser.tenantId;

  if (loginUser.isAdmin || loginUser.roleKey === 'superadmin') {
    return String(currentTenantId || '').trim();
  }

  return loginUser.tenantId;
}
```

重要：

```text
普通用户永远使用自己的 tenant_id。
超管可以使用当前选择的租户上下文。
超管不传 currentTenantId 时，默认使用自己的 tenant_id。
```

## 3. searchIndexRepository.search

```ts
async function search(params: {
  tenantId: string;
  keyword: string;
  permissions: string[];
  roles: Role[];
  userId: string;
  limit: number;
}) {
  const dataScope = buildDataScopeWhere(params.roles, params.userId);

  const sql = `
    SELECT
      index_id,
      module_key AS moduleKey,
      module_name AS moduleName,
      biz_id AS id,
      title,
      subtitle,
      route_path AS routePath,
      updated_at AS updatedAt
    FROM sys_search_index
    WHERE tenant_id = :tenantId
      AND deleted = 0
      AND status = '0'
      AND permission IN (:permissions)
      AND (
        title LIKE :keyword
        OR subtitle LIKE :keyword
        OR content LIKE :keyword
      )
      ${dataScope.sql}
    ORDER BY updated_at DESC
    LIMIT :limit
  `;

  return db.query(sql, {
    tenantId: params.tenantId,
    permissions: params.permissions,
    keyword: `%${params.keyword}%`,
    limit: params.limit,
    ...dataScope.params,
  });
}
```

如果当前数据库工具不支持 `IN (:permissions)` 数组参数，请改成动态占位符：

```ts
const permissionPlaceholders = permissions.map((_, index) => `:permission${index}`).join(',');

const permissionParams = Object.fromEntries(
  permissions.map((value, index) => [`permission${index}`, value])
);
```

## 4. groupSearchResults

```ts
function groupSearchResults(list: any[]) {
  const map = new Map<string, any>();

  for (const item of list) {
    if (!map.has(item.moduleKey)) {
      map.set(item.moduleKey, {
        moduleKey: item.moduleKey,
        moduleName: item.moduleName,
        routePath: item.routePath,
        list: [],
      });
    }

    const group = map.get(item.moduleKey);

    if (group.list.length < 5) {
      group.list.push({
        id: item.id,
        title: item.title,
        subtitle: item.subtitle,
        moduleKey: item.moduleKey,
        moduleName: item.moduleName,
        routePath: item.routePath,
      });
    }
  }

  return Array.from(map.values()).filter(item => item.list.length > 0);
}
```

不要返回：

```text
身份证
完整手机号
```

## 2. 动态字段必须校验

所有动态表名、字段名必须使用：

```ts
function safeIdentifier(value: string) {
  if (!/^[a-zA-Z0-9_]+$/.test(value)) {
    throw new Error(`非法字段或表名：${value}`);
  }

  return value;
}
```

普通用户即使传：

```text
x-tenant-id || currentTenantId || loginUser.tenantId
```

# 十六、验收标准

## 1. 普通用户

用户信息：

```text
tenant_id = 100001
permissions = exhibitor:search
```

搜索结果：

```text
只能看到 tenant_id = 100001 的展商结果。
不能看到用户管理、角色管理。
不能看到 tenant_id = 100002 的数据。
```

---

## 2. 租户管理员

用户信息：

```text
tenant_id = 100001
permissions = system:user:search, system:role:search, exhibitor:search
```

搜索结果：

```text
可以看到用户、角色、展商。
只能看到 tenant_id = 100001 的数据。
不能看到其他租户。
```

---

## 3. 平台超管

用户信息：

```text
tenant_id = 000000
permissions = *:*:*
```

当 `x-tenant-id = 100001`：

```text
只能搜索 tenant_id = 100001 的数据。
```

当 `x-tenant-id = 100002`：

```text
只能搜索 tenant_id = 100002 的数据。
```

当没有传 `x-tenant-id`：

```text
默认搜索 tenant_id = 000000 的平台数据。
不能默认跨租户搜索全部数据。
```

---

## 4. 搜索配置停用

当某个配置：

```text
enabled = 0
```

对应模块不参与全局搜索。

---

## 5. 权限删除

删除用户某个搜索权限后：

```text
清理 Redis 权限缓存。
再次搜索时，该模块不再出现。
```
