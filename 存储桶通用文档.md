# 存储桶通用文档

## 1. 架构概述

系统使用**策略模式 + 工厂模式**抽象多存储后端，通过 `sys_storage_config` 表配置，运行时动态选择 Provider。

```
upload → StorageFactory → Provider (Minio / AliyunOss / TencentCos / ...) → 存储服务
```

**当前实现状态**：仅 MinIO 完整实现，其余 Provider（AliyunOss、TencentCos、AwsS3、Local）为骨架占位。

## 2. 核心接口

所有 Provider 实现 `StorageProvider` 接口（`providers/StorageProvider.ts`）：

| 方法 | 说明 |
|------|------|
| `upload(params)` | 上传 Buffer，返回 `{bucketName, objectName, url?}` |
| `remove(params)` | 删除文件 |
| `getPublicUrl(params)` | 获取公开访问 URL |
| `getPrivateUrl(params)` | 获取带签名的私有 URL（支持过期时间） |
| `ensureBucket?(bucketName)` | 可选，自动创建 Bucket |

## 3. 数据库配置表 `sys_storage_config`

| 字段 | 说明 |
|------|------|
| `storage_type` | `minio` / `aliyun_oss` / `tencent_cos` / `aws_s3` / `local` 等 |
| `endpoint` | 服务端点 |
| `port` | 端口 |
| `use_ssl` | 是否 HTTPS |
| `access_key` / `secret_key` | 凭证 |
| `public_bucket` / `private_bucket` | 公开/私有桶 |
| `public_base_url` / `private_base_url` | 自定义域名 |
| `path_style` | Path Style 访问模式 |
| `config_json` | 厂商差异配置 |
| `policy_json` | 上传限制（maxSizeMB, allowedExt, blockedExt） |
| `is_default` | 默认配置标识 |

> 配置通过管理后台维护，非环境变量/配置文件方式。

## 4. MinIO 配置（唯一完整实现）

依赖 `minio` 包（v8.x），核心功能：

- `ensureBucket()`：自动检查并创建 bucket
- `upload()`：`client.putObject()` 真正上传
- `remove()`：`client.removeObject()` 真正删除
- `getPublicUrl()`：支持 `publicBaseUrl` 或拼接 endpoint
- `getPrivateUrl()`：`client.presignedGetObject()` 生成预签名 URL（默认 300s）

### 推荐配置

```json
{
  "storage_type": "minio",
  "endpoint": "127.0.0.1",
  "port": 9000,
  "use_ssl": false,
  "access_key": "minioadmin",
  "secret_key": "minioadmin123",
  "public_bucket": "bls-files",
  "private_bucket": "bls-files",
  "path_style": true
}
```

### 注意事项

- **pathStyle**：建议 `true`，否则部分部署环境下可能无法访问
- **HTTPS**：生产环境建议 Nginx 反代提供 HTTPS
- **桶权限**：私有桶由后端生成签名 URL，公开桶直接返回 URL

## 5. 扩展新存储提供商

1. 在 `providers/` 下新建 Provider 类，实现 `StorageProvider` 接口
2. 在 `storage.factory.ts` 的 `createStorageProvider()` 中注册新类型
3. 安装对应 SDK 依赖

## 6. API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/system/storage/list` | 存储配置列表 |
| POST | `/system/storage/add` | 新增配置 |
| PUT | `/system/storage/edit` | 编辑配置 |
| DELETE | `/system/storage/remove` | 软删除配置 |
| POST | `/system/storage/upload` | 文件上传（限流 30次/分钟） |
| GET | `/system/storage/files` | 文件列表 |
| DELETE | `/system/storage/file/:fileId` | 删除文件 |
| GET | `/system/storage/file/:fileId/url` | 获取文件 URL |
| GET | `/system/storage/file/:fileId/download` | 文件下载 |

## 7. 上传流程

1. 接收 multipart 文件 → 解析 `accessType`、`storageId`
2. 查 `sys_storage_config` 获取配置（优先指定 storageId，否则取 `is_default=1`）
3. `createStorageProvider(config)` 获取 Provider
4. 读取文件 Buffer，调用 `provider.upload()`
5. 雪花 ID 写入 `sys_file` 表 → 返回 `{fileId, url, bucketName, objectName, ...}`
